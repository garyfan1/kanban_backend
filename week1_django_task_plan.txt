WEEK 1 — DJANGO BACKEND TASK PLAN

GOAL: By the end of Week 1, you will have a secure Django REST API that
supports authentication and full CRUD operations for user-owned tasks.

Daily time target: 1.5–3 hours max

  --------------------------------
  DAY 1 — Project & Environment
  Setup
  --------------------------------
  - Create and activate virtual
  environment - Upgrade pip -
  Install dependencies: - django -
  djangorestframework -
  django-cors-headers - Start
  Django project - Run development
  server - Create apps: -
  accounts - tasks - Add apps to
  INSTALLED_APPS - Setup .env and
  move SECRET_KEY - Create
  superuser - Verify Django admin
  works

  DONE WHEN: - Server runs without
  error - Admin panel accessible
  --------------------------------

DAY 2 — DRF & CORS Setup

-   Add Django REST Framework to INSTALLED_APPS
-   Configure REST_FRAMEWORK settings:
    -   Default permission: IsAuthenticated
    -   Default authentication: SessionAuthentication
-   Add django-cors-headers
-   Configure middleware order correctly
-   Allow localhost origins
-   Create a simple health-check endpoint: GET /api/health/
-   Return JSON response

DONE WHEN: - API returns JSON - Permissions enforced - Health endpoint
works

  ---------------------------------
  DAY 3 — Authentication (JWT)
  ---------------------------------
  - Install
  djangorestframework-simplejwt -
  Configure JWT authentication in
  settings - Set access & refresh
  token lifetime - Add auth
  endpoints: - POST
  /api/auth/login/ - POST
  /api/auth/refresh/ - Create users
  via Django admin - Test auth flow
  using Postman or curl: - Login -
  Access protected endpoint -
  Refresh token

  DONE WHEN: - JWT login works -
  Unauthorized requests are blocked
  ---------------------------------

DAY 4 — Task Model & Admin

-   Create Task model with fields:
    -   user (ForeignKey)
    -   title
    -   description
    -   status (todo / doing / done)
    -   created_at
    -   updated_at
-   Add str method
-   Create and run migrations
-   Register Task in admin
-   Customize admin list display
-   Create sample users and tasks
-   Verify task-user isolation

DONE WHEN: - Admin CRUD works - Tasks correctly belong to users

  --------------------------------
  DAY 5 — Task CRUD API
  --------------------------------
  - Create TaskSerializer -
  Validate title and status -
  Implement endpoints: - GET
  /api/tasks/ - POST /api/tasks/ -
  PATCH /api/tasks/{id}/ - DELETE
  /api/tasks/{id}/ - Ensure
  querysets are filtered by
  request.user - Enforce
  object-level permissions - Test
  all endpoints thoroughly

  DONE WHEN: - Full CRUD works -
  Users cannot access others’
  tasks
  --------------------------------

DAY 6 — Polish & Hardening

-   Add pagination
-   Add ordering (created_at, status)
-   Optional status filtering
-   Standardize error responses
-   Remove unused code
-   Add comments where logic is non-obvious
-   Full smoke test from scratch database

DONE WHEN: - Backend feels complete - No TODOs left - Ready for frontend
integration

  ---------------
  END OF WEEK 1
  ---------------

You now have: - A modern Django REST backend - JWT authentication -
Secure, user-scoped CRUD APIs
